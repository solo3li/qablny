# 📱 تحليل تطبيق **قابلني** (Qablny) - الواجهة الأمامية (Frontend)

> تطبيق تعارف ومحادثات مرئية مستوحى من Azar — مبني بـ React Native / Expo للويب والموبايل

---

## 🧱 Stack التقني

| الطبقة | التقنية |
|--------|---------|
| Framework | **React Native 0.81.5** + **Expo ~54** |
| Navigation | **expo-router ~6** (File-based routing) |
| State Management | **Zustand ^5** |
| Real-time Signaling | **SignalR Client** |
| Video/Audio Calls | **LiveKit React Native** (`@livekit/react-native`) |
| API Requests | **Axios** (`axiosClient.ts`) |
| UI/Icons | **lucide-react-native**, **expo-blur**, **expo-linear-gradient** |

---

## 📡 الربط مع الخوادم (Backend Integration)

التطبيق لم يعد يعتمد على البيانات الوهمية (Dummy Data) في الجزء الخاص بالمحادثات، بل تم ربطه بشكل كامل مع البنية التحتية للخوادم:
1. **SignalR (`chatSignalR.ts`):** 
   - يدير اتصال الـ WebSockets لتحديثات الزمن الفعلي.
   - يعالج أحداث `ReceiveMessage`, `MessageSent`, `TypingStarted`, `UserOnline`, `IncomingCall`, `CallAccepted` وغيرها.
2. **الـ Store (`chatStore.ts` و `callStore.ts`):**
   - **`chatStore`:** يحفظ حالة الرسائل، والمحادثات، ويربط الـ `BackendChatMessage` بـ `UIChatMessage`. يدعم الـ Optimistic Updates.
   - **`callStore`:** يدير حالات مكالمات LiveKit (الغرفة، التوكن، حالة المكالمة النشطة والواردة).
3. **مكالمات WebRTC:**
   - استخدام LiveKit لإنشاء غرف الاتصال فور الموافقة على المكالمة، سواء كانت صوتية أو مرئية، بشكل Peer-to-Peer حقيقي.

## 📺 الشاشات والتحديثات الأخيرة

### 1. `chat/[id].tsx` — شاشة المحادثة المتقدمة
- أصبحت تعتمد على `useChatStore` لجلب الرسائل الحقيقية المرتبطة بـ `friendId`.
- **مؤشرات الزمن الفعلي:** إظهار "يكتب الآن..."، "متصل"، "يسجل مقطع صوتي...".
- **أنواع رسائل مدعومة بالكامل:** Text, Audio (Voice), Video, Image, Location.
- **سجل المكالمات (Call Logs):** ظهور رسائل مخصصة في الشات (مكالمة لم يرد عليها 🟥، مكالمة صوتية 📞 (مع مدتها)، مكالمة فيديو 🎥) وتُرسل تلقائياً من الباك إند عند إنهاء المكالمة أو رفضها، لتجربة شبيهة بـ Telegram.

### 2. `components/CallModal.tsx` — مودال المكالمات
- تم ربطه بـ **LiveKit**.
- يستقبل إشعارات SignalR لبدء الرنين. عند القبول، يقوم بالانضمام للغرفة الحقيقية عبر `livekitApi.ts`.
- يقوم بإرسال إشارة الرفض (Decline) لإنهاء المكالمة في الطرف الآخر فوراً لمنع تعليق واجهة الاتصال (Call Ghosting).

### 3. `messages.tsx` — قائمة المحادثات والأصدقاء
- تعرض قائمة الأصدقاء والمحادثات النشطة بناءً على الاستجابات الفعلية من الخادم.
- عرض فوري لعدد الرسائل غير المقروءة وأحدث رسالة مستلمة.

## 🎨 نظام التصميم (Glassmorphism)
- استمرار استخدام الـ Glassmorphism كسمة رئيسية للتطبيق (`#070B14` كخلفية، مع `rgba(255,255,255,0.06)` للـ Surface).
- استخدام الألوان المخصصة للإشارات (Indicators): `Colors.online` (الأخضر)، `Colors.danger` (الأحمر).

## ✅ نقاط تم إنجازها
1. **استبدال الـ Dummy Data بخادم حقيقي** في وظائف الدردشة الأساسية وإدارة الأصدقاء والمكالمات.
2. **مكالمات حقيقية** بدلاً من الواجهة الوهمية باستخدام تقنية LiveKit.
3. **تزامن حالات المستخدم** (Online, Typing, Recording) بشكل لحظي ودقيق.
4. **سجل مكالمات احترافي** يوثق المكالمات داخل المحادثة بشكل ديناميكي من الخادم للعميل.

## ⚠️ نقاط جاري العمل عليها / المتبقية
- **شاشات المطابقة (MatchView):** تفعيل وتطوير الـ Queue العشوائي وإظهار الكاميرا الحقيقية مع المستخدمين العشوائيين.
- **إدارة طلبات الصداقة (Friend Requests):** تصميم شاشة مخصصة (UI) لقبول أو رفض طلبات الصداقة وعرض إشعارات الطلبات الجديدة بدلاً من الإضافة التلقائية.
- **تعديل وحذف الرسائل:** الأكواد الأساسية للخادم موجودة بالفعل، يتبقى ربطها بالكامل بواجهة المستخدم وربطها بقائمة السياق (Context Menu).
- **إرسال الهدايا الافتراضية:** تفعيل التأثيرات الحية (Animations) عند إرسال الهدايا وإظهارها للطرف الآخر أثناء الاتصال.
